# Treat client artifacts and coverage profiles as products, not versions

The repository publishes each verified source snapshot by updating the fixed Client Artifacts under `main/dist`; it does not create version tags or GitHub Releases. Strict and maximum coverage are named profiles of the same Canonical Rule Model, while each client format is a Client Artifact rather than a competing version. Shadowrocket artifacts are generated from that same model. The module remains the safe default because it does not replace the user's existing configuration; the additional full-configuration installation choice is recorded in ADR 0002.
