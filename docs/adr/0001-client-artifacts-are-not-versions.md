# Treat client artifacts and coverage profiles as products, not versions

The repository has one release identity per verified source snapshot. Strict and maximum coverage are named profiles of the same Canonical Rule Model, while each client format is a Client Artifact rather than a competing version. Shadowrocket receives a separately installable Standalone Complete Module generated from that same model; a module deep link was chosen over importing a full configuration because it can add the complete rule product without replacing the user's nodes, groups, DNS settings, or final policy.
