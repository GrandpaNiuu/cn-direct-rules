# Domain docs

This repository uses a **single-context** layout.

Before changing rule meaning, source policy, coverage profiles, or generated client artifacts, read:

- `CONTEXT.md` for the repository's canonical domain language;
- relevant decisions in `docs/adr/`.

Use the glossary's preferred terms in issues, tests, documentation, and code. If a proposed change conflicts with an ADR, surface the conflict explicitly instead of silently overriding the decision.
